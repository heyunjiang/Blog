# 面试引导

time: 2021-10-09 10:20:26  
author: heyunjiang

## 1 背景

最近在面试外包伙伴时，遇到一个问题：简历上写了很多技术名次，比如熟悉 vue, vue-router, webpack 等，但是一问只会基础使用，大体配置都说不清楚。  
这对于面试官的印象不会太好

还有一个方面，自己在面试时，也会出现不知道该怎么去引导面试官问我擅长的领域，自己擅长的也不太多。

## 2 面试原则

1. 展示自身能力：把自己擅长的技术点、沟通能力、解决问题能力体现出来
2. 重点引导：着重提示自身会的点，比如擅长 vue 项目开发、系统性能优化

## 3 面试套路

通过举证来体现自身能力  
1. 说明自己擅长某个方向：擅长 vue 系统开发，能解决绝大部分 vue 相关问题
2. 问题举例：之前遇到过 ref 绑定后在组件内部通过 this.$refs 读取不到的问题
3. 解决问题的思路：首先查看官方文档教程及 Q&A，再 google 查询，最后通过源码验证或者调试定位问题。
说出关键源码词：template 编译为 render 函数，ref 作为 createElement 第二个属性参数；
生成 vnode 之后，通过 vm._update() 渲染时，会调用 registerRef 来为 vm.$refs 添加 ref 属性；
通过 initRender 为 vm.$createElement 绑定的 vnode.context = vm 属性
4. 文档总结：总结相关问题，写成文档在公司内部共享、github 分享记录

## 4 日常准备

1. 日常遇到 vue 相关问题直接上，这是提升自己实例的核心点，遇到问题、解决问题、总结问题
2. 形成 vue 系统思维，知道 vue 项目如何组织代码、new Vue 实现的过程、vue 有哪些隐藏的坑、核心模块有哪些、学习资源方式、vue3 未来发展方向

## 5 我擅长的方向总结

1. 原生基础：js、dom、bom、chrome-dev-tool、浏览器基础原理、网络原理
2. 核心能力：缓存、浏览器内存管理、抓包与分析、系统性能优化、构建及性能优化
3. 框架原理：vue2，vue3 及全家桶、webpack 基础构建流程、loader 和 plugin 原理、webpack module federation
4. 解决问题能力：套路总结
5. 管理能力：带前端团队，负责众多跨部分前端项目管理，主要体现在研发流程
